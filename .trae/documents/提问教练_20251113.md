# 提问教练 Prompt（专业版）

日期：2025-11-13

## 概览与目标
- 引导用户进行批判性思考，结构化拆解问题，形成稳健结论与行动方案。
- 适配不同知识水平（新手/进阶/专家），提供逐层递进的提问框架。
- 基于“12步批判性提问框架”与价值观指导（好奇心、开放性、证据导向、谦逊）。

## 使用前输入
- 议题与背景：[简述场景、涉众、时间线]
- 目标与约束：[期望结果、成本/风险、期限]
- 当前主张：[你或团队的初步结论]
- 可用证据：[数据、研究、专家意见、案例]

## 分层提问框架

### 第一层：问题澄清（适用于新手，提升清晰度）
- 论题类型：这是“是什么/为何”还是“应不应该/如何做”的问题？
- 核心结论：目前的最终主张是什么？是否表述明确、可检验？
- 支持理由：每个关键主张背后的理由是什么？逻辑链是否完整？

### 第二层：论证分析（进阶，提升严密性）
- 概念清晰性：关键术语定义是否一致？是否存在多义或情感化用词？
- 隐藏假设：价值观假设与事实性假设分别是什么？它们如何影响结论？
- 推理谬误：是否存在循环论证、稻草人、错误类比、相关性当因果等？

### 第三层：证据评估（进阶，提升可靠性）
- 证据来源：直觉/经验/案例/专家意见的可靠性与相关性如何？
- 研究证据：样本是否具代表性？方法与结论是否匹配？控制变量是否充分？
- 替代原因：是否存在其他合理解释？多重原因是否被系统评估？

### 第四层：数据与完整性（高阶，提升健壮性）
- 数据质量：来源、采集过程、统计口径是否透明？平均值与百分比是否恰当？
- 信息遗漏：有哪些关键未被讨论的信息？潜在负面影响是否被考虑？
- 完整呈现：不同方案的利弊是否对称呈现？不确定性是否被明确标注？

### 第五层：结论构建与决策（高阶，提升可执行性）
- 结论形态：二元对立之外，是否可以形成条件化/情境化结论？
- 决策规则：在何种条件下采纳/暂缓/否决？需要哪些监控指标与触发点？
- 行动方案：下一步行动、负责人、时间节点、验证方法、回滚预案。

## 核心提问清单（可直接复制使用）
- 开场：你希望解决什么问题？为什么重要？谁受影响最大？
- 结构：你的最终主张是什么？有哪些关键理由支撑？
- 概念：哪些术语容易混淆？我们采用的定义是什么？
- 假设：背后的价值观与事实假设分别是什么？它们是否成立？
- 谬误：推理链条里可能的漏洞是什么？
- 证据：证据的来源、质量与适用边界是什么？
- 替代：还有哪些合理的替代解释或方案？
- 风险：最坏情况是什么？如何提前识别与应对？
- 结论：在什么条件下我们的结论才成立？如何监控与校正？
- 行动：下一步要做什么？谁负责？何时验证结果？

## 通用 Prompt（复制即用）

### 模板结构说明
- 问题定义：明确论题类型、核心结论与评估目标。
- 上下文说明：背景、涉众、约束（时间/风险/资源）、已有证据。
- 输出要求：结论形态、决策条件、行动方案、监控与复盘要点。

### 通用模板
```txt
[目标]
- 论题类型：{{topic_type}}
- 评估目标：{{assessment_goal}}
- 结论期待：{{expected_conclusion}}

[上下文]
- 背景：{{background}}
- 涉众与影响：{{stakeholders}}
- 约束（时间/风险/资源）：{{constraints}}
- 现有证据与数据口径：{{evidence}}

[分析要求]
- 术语定义与关键假设清单
- 谬误检查与替代解释对比
- 证据质量评估（来源/方法/匹配度）

[输出要求]
- 条件化/情境化结论与采纳规则
- 行动方案（负责人/里程碑/时间线）
- 监控指标与回滚预案
- 风险与负面效应的对称呈现
```

### 变量与使用说明
- `{{topic_type}}`：论题类型，如“是否上线”/“因果关系评估”。
- `{{assessment_goal}}`：衡量目标，如“提升 D30 ≥ 5%”。
- `{{expected_conclusion}}`：期望输出的结论形态，如“条件化采纳规则”。
- `{{background}}`：业务/研究背景与触发原因。
- `{{stakeholders}}`：涉众及影响范围（用户/客户/团队/监管）。
- `{{constraints}}`：时间/预算/合规/技术等约束与边界。
- `{{evidence}}`：已有数据与来源、统计口径与有效期。

### 错误处理与使用限制
- 避免以相关性替代因果；补充替代解释并进行竞争性比较。
- 明确统计口径与样本代表性；不足时标注不确定性并降级结论强度。
- 输出必须以“条件化陈述”呈现，并给出可执行的采纳/回滚规则。
- 人工复核关键假设与高影响风险；模型结论不得直接用于生产决策。

### 示例 1：产品功能上线评估
```txt
[目标]
- 论题类型：是否在下季度上线“智能推荐”功能（规范性 + 行动性）
- 评估目标：在不提升投诉率的前提下，D30 ≥ 5%，ARPU ≥ 3%
- 结论期待：给出条件化结论与采纳/回滚规则

[上下文]
- 背景：核心留存承压；竞品近期上线相似功能并有正向反馈
- 涉众与影响：现有用户、新用户、内容团队、风控与客服
- 约束（时间/风险/资源）：两周埋点校验，四周 A/B 测试；人力有限
- 现有证据与数据口径：访谈与原型实验，ARPU 按自然周口径

[分析要求]
- 术语与假设：D30 定义；用户对个性化敏感；推荐准确率 ≥ 60%
- 谬误与替代：相关性≠因果；节日/渠道活动可能导致指标提升
- 证据评估：竞品案例存在情境差异；专家意见需结合本域数据

[输出要求]
- 结论：当 D30 ≥ 5%、ARPU ≥ 3%、投诉率不增时采纳，否则回滚
- 行动：两周埋点校验 → 四周 A/B → 结果复盘；负责人与时间线明确
- 监控与回滚：异常告警阈值与撤回流程预设；记录影响面与复盘要点
- 风险：算法偏见与信息茧房；用户信任受损的缓释方案
```

### 示例 2：论文/研究报告评估
```txt
[目标]
- 论题类型：研究结论可信度与可迁移性评估
- 评估目标：判断方法与结论匹配度、样本代表性与外部有效性
- 结论期待：给出采纳条件、限制边界与后续验证计划

[上下文]
- 背景：拟采用某论文的算法作为生产基线
- 涉众与影响：产品团队、数据平台、合规与隐私监管
- 约束：数据脱敏与最小化原则；算力预算受限
- 现有证据：论文公开数据集与实验代码；评测指标为 AUC/F1

[分析要求]
- 术语与假设：样本独立同分布；训练/测试不泄漏
- 谬误与替代：过拟合风险；数据集偏差导致的伪相关
- 证据评估：来源可信度、复现结果与统计显著性

[输出要求]
- 结论：当在自有数据上复现实验且差异 ≤ 3% 时方可采纳
- 行动：数据映射与口径统一 → 复现实验 → 迁移评估 → 安全检查
- 监控与回滚：线上灰度与异常阈值；影响评估与撤回机制
- 风险：隐私泄露与合规风险；黑盒难解释的缓解与审计方案
```

### 示例 3：公共政策争议话题分析
```txt
[目标]
- 论题类型：政策利弊与证据强度评估
- 评估目标：识别受益/受损人群与短中长期影响
- 结论期待：在不同情境下的条件化建议与监测指标

[上下文]
- 背景：某城市拟推行差异化交通收费政策
- 涉众与影响：居民、通勤者、商户、环保组织、财政部门
- 约束：财政预算、法规与社会接受度
- 现有证据：试点数据与第三方研究报告

[分析要求]
- 术语与假设：拥堵定义与测度；价格弹性与替代交通工具
- 谬误与替代：选择性统计与忽略负外部性；“后此因彼”错误
- 证据评估：来源中立性、方法适用性与结论边界

[输出要求]
- 结论：在拥堵指数 ≥ X 且公共交通承载能力 ≥ Y 时实施为宜
- 行动：分阶段试点与利益相关方沟通；定期评估与调整机制
- 监控与回滚：拥堵/排放/民意指标的阈值与回滚条件
- 风险：行业结构性冲击与弱势群体受损的补偿方案
```

### 示例 4：项目复盘与改进计划
```txt
[目标]
- 论题类型：失败原因识别与改进优先级排序
- 评估目标：构建可执行的改进路线图与监控指标
- 结论期待：提出 2–3 项高杠杆改进与验证计划

[上下文]
- 背景：版本发布延期且用户反馈不佳
- 涉众与影响：研发、产品、运营、合作方
- 约束：人力与预算有限；上线窗口固定
- 现有证据：需求变更记录、缺陷统计、性能监测数据

[分析要求]
- 术语与假设：缺陷密度与优先级定义；瓶颈定位方法
- 谬误与替代：以个案替代总体；忽略上下游依赖的影响
- 证据评估：数据质量与覆盖度；因果链条的可验证性

[输出要求]
- 结论：以“交付节拍稳定性”为首要目标，制定分期改进计划
- 行动：需求冻结机制 → 回归测试完善 → 性能瓶颈整改
- 监控与回滚：节拍/缺陷/性能三类核心指标，异常触发回滚或延期
- 风险：变更管理失败与技术债积累；设定缓冲与熔断规则
```

## 输出模板（交付物标准）
- 问题澄清：论题类型、核心结论、关键术语与定义。
- 论证分析：理由结构图、假设清单、可能谬误。
- 证据评估：证据来源矩阵、质量与边界说明、替代解释比较。
- 决策结论：条件化/情境化结论、采纳/暂缓/否决规则。
- 行动方案：步骤、负责人、时间线、验证指标、回滚预案。
- 复盘要点：结果与假设对照、偏差来源、后续迭代建议。

## 适配不同知识水平
- 新手版：优先使用“问题澄清+概念清晰+基本证据”三件套，避免陷入细节。
- 通用版：覆盖“12步框架”的关键节点，保证论证与证据的最小充分性。
- 专家版：引入不确定性标注、贝叶斯更新、反事实推理与敏感性分析。

## 时间盒流程建议
- 10分钟：问题澄清与概念统一。
- 20分钟：理由梳理、假设识别、初步证据评估。
- 15分钟：替代解释与风险研讨。
- 15分钟：结论与行动方案定稿；明确监控与复盘机制。

## 注意事项
- 保持中立，不预设结论；以证据与逻辑为准绳。
- 鼓励自我质疑与反思，避免过度自信与确认偏误。
- 在团队讨论中，确保不同观点被充分表达与记录。
